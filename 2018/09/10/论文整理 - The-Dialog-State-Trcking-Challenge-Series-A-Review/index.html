论文整理 - The Dialog State Trcking Challenge Series: A Review

发表于 2018-09-10

这篇文章最早发表在AAAI2014，这篇是2016年的版本，增加了一些2014-2016期间的相关研究进展。

对话状态跟踪(DST)是给定所有对话历史，正确推断对话状态（如用户目标）的任务。

对话状态s_t是从集合S中获取的数据结构，它将直到t时刻的对话历史归结起来，并为选择下一个系统动作提供足够的信息。在实践中，对话状态通常编码了用户目标和对话中的相关历史。

对话状态跟踪器(DS Tracker)将直到t时刻的所有可观测元素和外部知识源作为输入。给定这些输入，DS Tracker输出对当前对话状态s的估计，目标是正确识别对话中的状态s*。一个鲁棒的DS Tracker应产生多个可能的对话状态的分布。

文章将对话状态看作是由人类可解释的元素组成，进一步假设DS Tracker能够产生对话策略(DP)模块的关键输入。DP则基于当前的对话状态产生行为或者响应。

这篇文章主要涉及了三种对话状态跟踪的方法，分别是：基于手工规则的方法、基于生成式模型的方法和基于判别式模型的方法。下面分别介绍。

基于手工规则 Hand-crafted Rules

基于手工规则的方法通常只考虑一个SLU结果，并跟踪单个对话状态假设。这种设计将对话状态跟踪问题简化为一个更新规则————根据上一个存在的对话状态s和一个SLU结果u来推导一个新的对话状态s’。

基于手工规则的方法的好处是他们不需要任何数据来实现，因此也易于做bootstrap。规则也使得开发者能够很好的将领域知识进行结合。

跟踪单个对话状态的基于手工规则的方法的一个缺点是他不能够使用完整的ASR/NLU的N-Best列表。因此近年来的基于手工规则方法的DS Tracker会根据整个ASR/SLU的N-Best列表计算所有对话状态的分数。这些方法使用手工设计的公式，根据ASR和SLU给出的置信度分数和先验估计，计算对话状态s的后验估计b(s)。

这种方法的主要缺陷是：计算公式的参数不是直接从真实对话数据中直接推导出来的，因此他们需要仔细调整并且不能够从对话数据中学习或获益。因此我们需要采用数据驱动的技术来自动设置参数以最大化准确率。因此就有了基于生成和基于判别的方法。

生成式模型 Generative Models

生成式的方法假定对话能够建模为一个贝野斯网络，该网络将系统行为a、用户行为u及ASR/SLU结果与对话状态s进行关联。当系统行为和ASR/SLU结果被观测到时，可能的对话状态的分布就能够通过贝叶斯网络推导得到。

公式1提供了其中的一种思路，此外还存在多种变体。这些方法的关键假设是可能的对话状态的分布能够使用一个贝叶斯网络进行推导，这个网络编码了对话的知识。模型的参数必须被估计，能够通过标注数据进行训练或利用未标注的对话数据通过最大期望或期望传播进行推导。

在端到端评估中，生成式模型比手工规则表现出更好的结果。但生成式模型不能轻易将ASR/SLU及对话历史中的潜在信息特征进行合并。所有的依赖关系需要被明确建模。而这一过程需要大量数据。因此，为了方便处理，生成模型通常采用一些无效的独立假设，或者违反了马尔科夫假设从而忽略对话历史中的某些重要特征。

这些缺陷又促使人们研究训练直接进行判别的模型方法。

判别式模型 Discriminative Models

判别式模型的主要优点是他们可以包含大量特征，并能够直接根据预测准确率进行优化。

常见的方法通过将对话历史编码为特征以学习一个简单的分类器。此外，还存在一些可以将对话建模为顺序过程的方法。包括：判别式马尔可夫模型、条件随机场或循环神经网络。

同样也存在基于循环神经网络的工作，直接使用ASR的输出而跳过了SLU模块。这些方法使得对话模型不需要进行特征设计，去除了忽略重要特征的风险，但与此同时可能会降低性能。

基于判别式模型的方法都需要用于训练的领域内的对话数据。当目标领域只存在少量标记数据时，需要采用跨领域迁移学习的方法。如果不存在标记数据，则可以使用基本模型对相关域进行无监督调整。

研究前沿 State-of-the-art

文章对DSTC1/DSTC2/DSTC3的结果进行比较，总结出当前（当时）最前沿的一些研究结果：

  • 判别模型 Discriminative models：判别式模型在几个测试中的性能都高于基于规则/基于生成的模型；
  • 使用ASR特征 Use ASR Feature：最好的DS Tracker总是包含低级别的ASR特征。ASR结果如N-best分数和单词困惑网络分数有利于提高精确度，且结合ASR结果会产生额外的对话状态假设进而提高召回率。
  • 序列化 Sequential：最好的DS Tracker要么讲对话直接建模为序列，要么集成了众多对话历史中的特征。像规则方法或基于生成的方法那样只考虑单一轮次的对话状态，效果不佳。
  • 捕获特征交互 Capture feature interactions：最好的DS Tracker直接建模特征之间的交互，如采用神经网络或决策树集合。
  • 联合后验：最好的系统是在所有槽值上计算联合后验，而不是将后验计算为每个槽值上margin的乘积。
