HCI论文共读 - 情感原因发现（试读版）

本月份起我组将举行月度论文共读活动，每月份组员阅读一定的学术文献并整理出笔记进行分享。2017年11月的论文阅读主题为“情感原因发现”，共涉及学术论文8篇。本版本为试读版本，主要用于测试笔记整理格式和邮箱推送情况。完整版本将在11月底左右进行推送。推送范围包括：HCI组成员邮箱和IBM实验室其他NLP小组成员邮箱。

文章索引：

  1. Emotion Cause Detection with Linguistic Constructions笔记 (By: R Zhang)
  2. A Question Answering Approach to Emotion Cause Extraction笔记 (By: K Yin)

Emotion Cause Detection with Linguistic Constructions

Authors: Chen, Y., Lee, S. Y. M., Li, S., & Huang, C. R.
Paper URL: https://dl.acm.org/citation.cfm?id=1873802
Year: 2010
Notes Author: R Zhang

1 问题定义

了解情感的存在并不足以预测未来事件或决定最佳反应，然而引发这些情感的原因一旦被发掘出来，对未来事件的预测或潜在影响的评估可以更可靠地完成。情感原因是指唤起相应情绪的因素，它们可以通过动词、名词或名词化词语来表达。
现有的原因检测系统主要将原因检测看作是对一个小范围的文本单元（通常是子句或者短语）对的提取，通常是｛情感类型，情感原因子句/短语｝的形式。这种方法不能够检测由多个子句所构成的情感原因。本文将情感原因发现转化为多标签分类问题，这种方法可以捕获远距离的文本信息。

2 语料分析

文章首先对(Lee et al, 2010)提出的中文情感原因语料进行了分析，发现72％的标注项中表达了情感信息，其中80%的情感信息具备一个情感原因，而且14%的情感原因由包含多个语句的复杂事件构成。文章进一步对情绪原因进行分析，发现情绪原因通常以动词形式表达(85%)而不是名词形式(15%)出现。
另外20%的抽取项找不到对应情感原因主要是因为：

  1. 抽取项在文段的开头或结尾；
  2. 上下文信息意义不大（如“啊哈？”）；
  3. 情感原因非常抽象。
    作者利用分隔符号（逗号、句号、问号、感叹号等）对篇章级文本进行划分：其中Left_1为情感词所在子句前的第一个子句，Right_1为情感词所在子句后的第一个子句，其他情况依此类推。Left_0表示从情感词所在子句开始直到情感词位置的部分子句，Right_0情况相反。依照上述划分方法，对情感原因统计得到：

3 方法描述

在原因关系识别领域，通常可以将该问题划分为两步进行：1）原因候选项识别；2）原因关系检测。为了简化任务，大多数方法跳过了第一步，通常基于领域知识通过一些预定义的过滤器对候选项进行过滤。在本文中，第一个要解决的问题是原因候选项的识别。
给定一个情感词及其上下文，该情感词的标签为其情感原因的位置信息，例如“Left_1, Left_0”。文章提出基于此构建多标签分类模型。这种方法具有两个优点：首先它能够从上下文信息中识别情感原因；其次它能够捕获多个候选情感原因之间的关系(例如，Left_2和Left_1可以合并为一个复杂事件作为原因)。

文章使用两种特征作为分类特征，一是基于模式的特征，二是基于匹配的特征。

3.1 语言线索和语言模式（基于规则）

文章提出了6种语言识别线索，列举如下：

类别 线索词
I. 介词 为、对、对于、以
II. 连词 因、因为、由于、于是、所以、因而、可是
III. 使动 让、令、使
IV. 动词 想到、想起、一想、想来、说到、说起、一说、讲到、
讲起、一讲、谈到、谈起、一谈、提到、提起、一提
V. 认知标记 听、听到、听说、看、看到、看见、见到、见、
眼看、瞧见、知道、得知、得悉、获知、获悉、
发现、发觉、有
VI. 其他 的是、地说、于、能

基于上述语言线索，文章将规则定义如下：
C = Cause 原因
K = Emotion Keyword 情感关键词
B = Clauses before the focus clause 当前子句之前的子句
F = Focus clause/the clause containing the emotion verb 当前子句/包含情感关键词的字句
A = Clauses after the focus clause 当前子句后的子句

共采用了13种语言规则，列举如下：

序号 规则
1 i) C(B/F) + III(F) + K(F)
ii) C = the nearest N/V before I in F/B
2 i) IV/V/I/II(B/F) + C(B/F) + K(F)
ii) C = the nearest N/V before K in F
3 i) I/II/IV/V(B) + C(B) + K(F)
ii) C = the nearest N/V after I/II/IV/V in B
4 i) K(F) + V/VI(F) + C(F/A)
ii) C = the nearest N/V after V/VI in A
5 i) K(F) + II(A) + C(A)
ii) C = the nearest N/V after II in A
6 i) III(F) + K(F) + C(F/A)
ii) C = the nearest N/V after K in F or A
7 i) 越 C 越 K
ii) C = the V in between the two 越 in F
8 i) K(F) + C(F)
ii) C = the nearest N/V after K in F
9 i) V(F) + K(F)
ii) C = V+(an aspectual marker) in F
10 i) K(F) + 的(F) + C(F)
ii) C = the nearest N/V + 的 + N after 的 in F
11 i) K(B) + IV(B) + C(F)
ii) C = the nearest N/V after IV in F
12 i) IV(B) + C(B) + K(F)
ii) C = the nearest N/V after IV in B
13 i) C(B) + K(F)
ii) C = the nearest N/V before K in B

以上述的第一个规则为例：

  1. C(B/F) + III(F) + K(F)
  2. C = the nearst N/V before III in F/B

上述规则的意思是，原因C是当前子句或之前的子句中“让、令、使”这些词前面距离最近的一个动词或者名词，如：“他的举动令我非常不满”。假设在Left_1位置找到了相应的原因C，则标记特征为：left_1_rule_1=1。

3.2 一般模式

基于规则的方法通常具有较高的准确率，但覆盖度较低。因此文章提出更加通用的匹配模式，列举如下：

  1. 局部功能性构造：一些特定的功能词如“因为、由于”的周围会表达情感原因。对于一个特定的情感词，通过从右到左搜索Left_0直到找到一个名词或动词，这个动/名词和情感关键词之间的所有一元和二元词项都被抽取出来；
  2. 长距离连接构造：通过词性标注，抽取那些连词出现在候选项开头的子句；
  3. 通用行为和认知动词：对于类别IV和V进行扩展，如由“听”扩展出“听见、听闻”等。

4 总结

文章提出了一种基于多标签分类的情感原因发现的思路。文章没有涉及到具体的多标签分类算法，仅仅是指出可以利用多标签分类框架解决这一问题。尽管实验结果显示基于多标签的方法效果好于单标签方法，特别是对于情感因素包含在多个子句或较长的短语中时，模型的作用能够体现出来。但文章的整体贡献较为有限。另外，基于模板的特征提取对于情感原因识别还是具有比较突出的作用。

A Question Answering Approach to Emotion Cause Extraction

Authors: Gui, L., Hu, J., He, Y., Xu, R., Lu, Q., & Du, J.
Paper URL: https://arxiv.org/abs/1708.05482
Year: 2017
Notes Author: K Yin

1 模型实现

本文采用了Memory Network来获取文档中一个字句和情感的相关关系。为了量化字句中单词和情感词汇的相关性,首先进行word embedding，以字句中单词和情感词汇的点积来度量相关关系的强弱；在通过softmax function将其统一到0和1之间，之后将字句中每个单词对应的词向量与对应的相关系数相乘并加和以此来度量该字句与情感词汇的距离，将此值与情感词汇向量E加和，再与待学习的参数矩阵相乘并通过softmax function来说明该字句描述的时间可否导致情感词汇描述的情感发生。
由于考虑到一个单词的具体意思通常是上下文相关的，为了表达这种关系，作者在原来的基础上加入了卷积的操作,将卷积核的大小设置为3，考虑每个单词和情感词汇的相关性时同时将单词的前一个词汇和后一个词汇考虑进去。最后在单层网络的基础上叠加了多层网络，利用随机梯度下降法进行学习。

2 总结

引入卷积的操作,对每个单词可以有效的考虑到其上下文的影响，对比孤立的考虑每个单词取得了更好的效果。

什么是终身机器学习(LML)？

传统的机器学习通常是孤立的单任务机器学习算法：针对某个特定任务，给定一份数据集，运行某些机器学习算法，训练得到一个模型。
这种传统的机器学习方法通常不会考虑到过去的知识。也就是说，每次遇到新的问题，传统机器学习方法都是从零开始训练一个新模型。这种方式被称为“机器学习1.0”。
“机器学习1.0”存在着一些明显的缺点，例如每次构建模型都是一次孤立学习过程，过往学习到的知识既没有被存储，也不能被重新获取。同时，也正因为知识不能够被存储，每次训练新模型时我们总是需要大量的新的训练数据。

Mac下卸载/更新numpy

2017-01-19

今天安装PyTorch的时候发现numpy模块需要更新，使用pip install -U numpy的时候发现有些操作被系统禁止了（不是sudo权限的原因）。百度了一下发现是系统的安全机制引起的。具体的解决方案如下：

在数据挖掘任务中，我们往往要先对数据进行预处理。本文介绍了数据处理的常见任务和常见方法。

数据预处理的主要任务

数据预处理的主要任务包括以下几个：

  1. 数据清洗：包括填入缺失值、平滑噪声数据、识别或移除离群点、解决数据不一致的状况

  3. 数据简化：如降维、数据块聚合（比如用聚类结果来替代原有数据）、数据压缩（比如利用编码减少数据长度等）
  4. 数据转换：包括归一化和泛化等等
